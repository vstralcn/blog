前置基础

1. alloc()、malloc()、calloc()、realloc()区别及用法

alloc

alloca是向栈申请内存,因此无需释放. alloc与afree以栈的方式（即后进先出的列表）进行存储空间的管理。



malloc()函数有一个参数,即要分配的内存空间的大小:     

　　void *malloc(size_t size); 



calloc()函数有两个参数,分别为元素的数目和每个元素的大小,这两个参数的乘积就是要分配的内存空间的大小。而且会初始化分配的内存空间
	void *calloc(size_t numElements, size_t sizeOfElement);


 realloc(void *__ptr, size_t __size)：更改已经配置的内存空间，即更改由malloc()函数分配的内存空间的大小。

==函数malloc()和函数calloc()的主要区别==是前者不能初始化所分配的内存空间,而后者能。如果由malloc()函数分配的内存空间原来没有 被使用过，则其中的每一位可能都是0;反之,如果这部分内存曾经被分配过,则其中可能遗留有各种各样的数据。也就是说，使用malloc()函数的程序开 始时(内存空间还没有被重新分配)能正常进行,但经过一段时间(内存空间还已经被重新分配)可能会出现问题。 需要调用函数memset来初始化这部分的内存空间.

2. brk()和sbrk()函数作用：

  • sbrk(increment)：用于改变程序的堆空间大小。increment 为正值时，增加堆空间；为负值时，减少堆空间；increment 为 0 时，返回当前程序断点的地址。需要注意的是，sbrk() 是一个不安全的接口，不应与其他内存管理函数（如 malloc()、free()）混用，以避免潜在的内存管理问题 。

  • brk(addr)：用于将程序的堆空间扩展或收缩到指定的地址 addr。如果 addr 大于当前程序断点，堆空间将增加；如果 addr 小于当前程序断点，堆空间将减少。与 sbrk() 类似，brk() 也属于不安全的接口，不应与其他内存管理函数混用。

3. mmap 函数原型

c

复制编辑

void *mmap(void *addr, size_t len, int prot, int flags, int fd, off_t offset);

  • addr：​建议的映射起始地址，通常设置为 NULL，让系统自动选择。

  • len：​映射区域的长度，必须是页面大小的整数倍。

  • prot：​映射区域的访问权限，使用按位或组合的方式指定，如 PROT_READ | PROT_WRITE。

  • flags：​映射的类型和行为，使用按位或组合的方式指定。

  • fd：​文件描述符，如果映射匿名内存，通常设置为 -1。

  • offset：​文件映射的起始偏移，必须是页面大小的整数倍。​sam4k

4. printf函数的缓冲区问题

printf是有缓冲区的，并不是直接将内容输出到屏幕上,它会先将需要输出的内容放入到缓冲区，直到缓冲区满或者遇到”\n”或者程序结束才将内容输出的屏幕上。

 printf 等函数，因为程序没有初始化缓冲区，如果引入这些函数的话，程序会创建一个堆用作缓存

5. 几种bin

bin（类似有分类的回收站）

由free chunk组成的链表，当用户再次申请时从中寻找合适的chunk返回给用户。按范围不同的chunk组成不同的bin如fast bin、small bin、large bin对应这些链表里面的chunk叫做fast chunk、samll chunk、large chunk。

bins：是bin的一个数组，共有126个bin、没有bin0

bin1：unsorted bin

bin2 ~bin63 ： small bin（62个）

bin64~bin126 ：large bin（63个）

各类bin

fast bins

用的比较多，一般情况下申请和释放的堆块都比较小，所以glibc对这类bin使用单链表结构，和stack一样也是先进后出的分配策略，fast 也就是快的意思。它里面的chunk不会进行合并操作和释放，下一个chunk的PRV_INUSE也就是p标志位恒为1，表示malloced状态，在fastbinsY数组中按小->大的顺序排序，序号为0的fast bin容纳4 * SIZE_SZ字节，之后随着逐个递增 2 * SIZE_SZ , 没有专属处理时不会被top chunk合并

unsorted bin

实际中一个释放的chunk很快就会被重用，所以先加入到unsorted bin可加快分配速度(类似缓存原理)，不同的是它是双链表结构（先进先出），是free chunk后会先加入到unsorted bin然后确认没有需求再回归对应bin区

small bin

和unsorted bin一样是双链表结构，每个small bin的大小为2 * SIZE_SZ * idx(bins数组下标)，因为它是bin2~bin63所有它在64bit中最小small chunk 为2 * 8 * 2 = 32byte，最大为1008byte

large bin

也是双链表结构，它在bins处于bin64~bin126，那么64bit中最小large bin为 2 * 8 * 64 = 1024byte，哎，这个时候前面说的fd、bk、fd_nextsize、bk_nextsize就来到用武之地了，为了加快检索速度，fd_nextsize和bk_nextsize指针会指向一个大小与自己不一样的chunk，故在加入了大小不同的chunk时，这两个指针才会被修改

程序运行时的整体内存布局（从低地址到高地址）

一、UAF

UAF即use after free，就是在程序使用malloc申请了内存并且写入了数据之后，通过free释放了堆块，但是没有清除数据，导致再次申请malloc时再次分配到这块内存，里面的数据被复用。（再次申请的堆块比之前申请的小）

主要是需要重新申请到之前存放函数指针的地址，覆盖之前写的数据，导致被误认为是以前的数据，从而控制程序

二、堆块重叠

原理：

通过堆溢出来对chunk的size字段进行修改，使得原来的两个堆块合并成一个堆块，然后就可以进行一些非法操作，例如泄露fd指针等…为什么能够实现？因为堆管理器只会检查chunk的nextsize字段是否合法，而nextchunk的地址获取是通过chunk+size获取，就会忽略中间的堆块，造成堆块合并

主要就是利用系统和实际的认知差来打，这个时候gdb不一定能准确识别出堆块的结构，请注意自己查看内存地址来确认分配

例题：pwn 141

三、 House of force

原理

针对top chunk进行攻击，通过这种攻击手法，将top chunk更新到任意地址，再次申请堆块并写入数据，相当于任意地址任意写
在libc2.23和2.27版本中，由于没有堆top chunk的size合法性检查，因此如果我们能够控制top chunk的size位以及malloc在申请堆块时的大小不受限制，就可以完成攻击
先从malloc函数源码看起，如果malloc函数执行时发现没有任何的bins中的堆块能够满足需求，就会从top chunk中切下一块内存返回给malloc**(前提是top chunk能够有这么多内存供切割)**

一般传统的house of force是通过把top chunck size改成一个很大的数字，常用0xffffffffffffffff。
当改好top chunck size后，我们就可以申请一个特别的chunck，让chunck覆盖到各种地方。
如，申请一个很大的chunck，使堆块覆盖到libc中的各种函数的hook部分，完成劫持。也可以申请一个负数，使堆块覆盖到got表的部分，完成got表的改写。

条件

libc2.23 - 2.29

1.需要能够控制top chunck的size
2.需要用户能自由分配malloc的申请大小

现在有物理空间连续的两个 chunk（Q，Nextchunk），其中 Q 处于使用状态、Nextchunk 处于释放状态。那么如果我们通过某种方式（比如溢出）将 Nextchunk 的 fd 和 bk 指针修改为指定的值。则当我们 free(Q) 时

  • glibc 判断这个块是 small chunk
  • 判断前向合并，发现前一个 chunk 处于使用状态，不需要前向合并
  • 判断后向合并，发现后一个 chunk 处于空闲状态，需要合并
  • 继而对 Nextchunk 采取 unlink 操作

那么 unlink 具体执行的效果是什么样子呢？我们可以来分析一下

  • FD=P->fd = target addr -12
  • BK=P->bk = expect value
  • FD->bk = BK，即 *(target addr-12+12)=BK=expect value
  • BK->fd = FD，即 *(expect value +8) = FD = target addr-12

看起来我们似乎可以通过 unlink 直接实现任意地址读写的目的，但是我们还是需要确保 expect value +8 地址具有可写的权限。

比如说我们将 target addr 设置为某个 got 表项，那么当程序调用对应的 libc 函数时，就会直接执行我们设置的值（expect value）处的代码。需要注意的是，expect value+8 处的值被破坏了，需要想办法绕过。
